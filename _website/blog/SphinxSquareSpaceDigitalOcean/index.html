Setting up a simple blog using sphinx and ablog using a SquareSpace domain, Nginx & DigitalOcean droplet#

Buying a domain on squarespace and droplet on Digital Ocean:#

  1. Buy a domain on squarespace.

  2. Create an Ubuntu droplet on Digital Ocean.

Installing Anaconda and building the website using sphinx and ablog:#

  1. Once you create the droplet above, ssh into it as root, create a separate user if you want and install anaconda.

  2. Once you install the conda environment, install the following python libraries via pip:

    pip install sphinx ablog pydata_sphinx_theme sphinx_panels

    Sphinx: A documentation engine written and used by the Python community.

    ablog: A Sphinx extension that converts any documentation or personal website into a blog.

    pydata_sphinx_theme: A simple, Bootstrap-based Sphinx theme made by the PyData community.

    sphinx_panels: A Sphinx extension for creating panels in a grid layout or as drop-downs.

  3. Create a source directory where all the configuration files will be placed.

  4. Run the following command in the terminal and answer the follow-ups as shown in the example

    $ sphinx-quickstart

accept a default value, if one is given in brackets).

"source" and "build" directories within the root path.

  5. The directory would look something like this:

    _build/

    _static/

    _templates/

    conf.py

    make.bat

    index.rst

    Makefile

  6. Open up conf.py and update the extensions & html_theme section as follows:

    extensions = [
"ablog",
"sphinx.ext.intersphinx",
"sphinx_panels",
]

html_theme = "pydata_sphinx_theme"

  7. Run ablog build to build the static pages. A directory called _websites will be created which will contain all the required htm, css and js files. The directory would look something like this at this stage:

    _build/

    _static/

    _templates/

    _website/

    conf.py

    make.bat

    index.rst

    Makefile

Setting up ngnix#

  1. Install nginx using the following command:

sudo apt install nginx

  1. The default directory for serving webpages is /var/www/html/. So we are going to move the entire _websites directory we created above to /var/www/html/.

  2. Open up the nginx conf using the command sudo nano /etc/nginx/sites-available/default and add /var/www/html/_website/ to the root section of the file in server block. It should look something like this:

# https://wiki.debian.org/Nginx/DirectoryStructure

    ...
}
